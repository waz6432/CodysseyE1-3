# 🚀 MAC 기반 패턴 매칭 및 성능 분석 시스템

본 프로젝트는 AI 연산의 핵심인 **MAC(Multiply-Accumulate)** 연산을 순수 파이썬으로 구현하고, 데이터 정규화 및 부동소수점 오차 정책을 통해 판정의 정확도를 높인 콘솔 애플리케이션입니다.

---

## 1. 실행 방법
* **환경:** Python 3.x (외부 라이브러리 설치 불필요)
* **명령어:** `python3 -m main.py`
* **모드 선택:** 
    - `1`: 사용자 직접 입력 ($3 \times 3$ 필터 A, B 및 패턴)
    - `2`: `data.json` 파일 로드 및 일괄 분석
* **주의사항:** `data.json` 파일이 `main.py`와 동일한 경로에 있어야 모드 2가 정상 작동합니다.

---

## 2. 구현 요약
* **MAC 연산:** `numpy` 등의 외부 라이브러리 없이 이중 `for` 루프를 사용하여 $N \times N$ 배열의 각 요소를 직접 곱하고 누적 합산하는 방식으로 구현했습니다.
* **라벨 정규화:** 원본 데이터의 `+`, `cross`, `x` 등 파편화된 라벨을 내부 표준인 `Cross`와 `X`로 매핑하여 판정 로직의 일관성을 확보했습니다.
* **동점 처리 정책(Epsilon):** 부동소수점 연산 오차를 고려하여, 두 필터 점수의 차이가 $1e-9$ 미만인 경우 `UNDECIDED`(판정 불가)로 분류하는 허용 오차 정책을 적용했습니다.

---

## 3. 결과 리포트 (분석 및 고찰)

### 3.1 실패 원인 및 성공 요인 분석
본 프로젝트의 `data.json` 분석 결과, 모든 테스트 케이스에서 **실패(FAIL) 0개**를 달성하였습니다. 이러한 안정적인 결과의 핵심은 데이터의 불확실성을 제어하는 '정규화'와 '수치 비교 정책'에 있습니다. 우선, 데이터 소스마다 상이했던 심볼들을 내부 표준 라벨로 통일함으로써 라벨 불일치로 인한 오판정을 차단했습니다. 또한, 단순 비교 연산자(`==`) 대신 $1e-9$ 수준의 **Epsilon 기반 비교 정책**을 도입하여 부동소수점 정밀도 문제를 해결했습니다. 만약 단순 비교를 사용했다면 하드웨어/OS 환경에 따른 미세한 연산 오차로 인해 동점 케이스가 FAIL로 처리될 수 있었으나, 허용 오차 정책을 통해 논리적 완결성을 높였습니다.

### 3.2 시간 복잡도 분석 ($O(N^2)$)
구현된 MAC 알고리즘은 패턴의 가로와 세로 길이를 각각 순회하는 이중 중첩 루프 구조를 가지므로 이론적인 시간 복잡도는 $O(N^2)$입니다. 실제 성능 측정 결과, $3 \times 3$($N^2=9$) 크기 대비 $25 \times 25$($N^2=625$) 크기에서 연산 횟수가 약 70배 증가함에 따라 실제 실행 시간 또한 이에 비례하여 비선형적으로 증가하는 것을 확인했습니다. 이는 I/O 부하를 제외한 순수 연산 구간만을 10회 반복 측정하여 얻은 데이터로, 알고리즘의 연산량이 데이터 크기의 제곱에 비례한다는 것을 실증적으로 증명합니다. 결과적으로 본 프로그램은 $O(N^2)$의 안정적인 성능 궤적을 유지하며 대규모 필터 환경에서도 예측 가능한 동작을 보장합니다.

## 5. 재현성 테스트 결과
* **모드 1:** $3 \times 3$ 십자가 필터와 X 필터 입력 시 점수 비교 및 판정(A/B)이 정상 출력됨을 확인.
* **모드 2:** `data.json` 로드 시 전체 테스트 수, 통과 수, 실패 수가 일치하며 PASS/FAIL 비교가 정확히 수행됨.
* **예외 처리:** 행/열 개수 불일치 또는 숫자 이외의 값 입력 시 안내 메시지 출력 후 재입력 유도 로직이 정상 작동함.
